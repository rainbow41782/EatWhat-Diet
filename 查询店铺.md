# 美团菜单爬取测试方式

现在的路线已经改成“人工养号浏览器 + 后端监听签名请求”，不再让 Selenium 或脚本去操作美团页面。

## 能导入哪些信息

- 店铺：店铺名、地址、经纬度、美团 `poi_id_str`、外部来源 `MEITUAN_WAIMAI`
- 菜品：菜品名称、分类、描述、价格、是否可售、美团菜品 ID
- 餐厅-菜品关系：餐厅 ID、菜品 ID、价格、规格/份量、SKU ID、图片链接备注
- 爬取任务：待查询、查询中、查询成功、查询失败、失败原因、导入菜品数、爬取次数、更新时间

## Cookie 怎么处理

不要把 cookie 填进 `application.yaml`，也不要在管理端页面里填写 cookie。

登录状态来自养号浏览器的本地资料目录：

```yaml
meituan:
  chrome:
    user-data-dir: "D:/projects/2026isd/JavaDiet/.chrome-meituan-profile"
    profile-directory: "Default"
    remote-debugging-port: 9222
```

也就是说，点“打开养号浏览器”后，在弹出的 Chrome 里人工登录美团即可。后端只监听这个浏览器已经成功发出的签名接口，不展示、不回传 cookie。

## 推荐测试流程

1. 启动后端和管理端。
2. 管理端登录后进入“美团菜单爬取”页面。
3. 点击“打开养号浏览器”。
4. 在弹出的 Chrome 中打开美团外卖首页，确认已经登录并且定位正常。
5. 回到管理端点击“开始监听签名请求”。
6. 在养号浏览器中手动搜索“麦当劳”，手动点进目标门店菜单页。
7. 菜单加载出来后，回管理端看“捕获接口”“菜单响应”“最近菜品”是否有数字。
8. 点击“导入最近捕获菜单”，后端会把刚捕获到的菜单写入 SQL。

## 批量任务怎么测

先完成一次人工监听，让页面显示“模板可用”。然后可以把待爬取店铺加入列表：

```text
麦当劳,北京市昌平区天通苑,40.069805,116.438572,B2X9-JMSWo_VPH81kcYmiQI,
```

点击“用模板爬取列表”后，后端会按列表顺序用最近捕获到的签名模板重放请求。若美团返回 403 或验证页，任务会显示“查询失败”，需要重新点击“开始监听签名请求”并人工打开一次店铺菜单来刷新模板。

最稳的测试方式是先只做“导入最近捕获菜单”。批量模板重放是否能跨店铺复用，取决于美团当前签名是否允许替换 `poi_id_str`。
